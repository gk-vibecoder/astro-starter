# i18n — Detailed Reference

> For the short mandate, see root `CLAUDE.md`. This file covers the full i18n architecture, layout adaptations for longer translations, and edge cases.

---

## Architecture

- Astro built-in i18n: `defaultLocale: 'en'`, `locales: ['en', 'sv']`, `prefixDefaultLocale: false`
- English at `/`, Swedish at `/sv/`
- `src/i18n/locales/en.ts` — all English UI strings
- `src/i18n/locales/sv.ts` — full translations (`satisfies typeof en` — TS build fails if any key missing)
- `src/i18n/utils.ts` — `getLangFromUrl(url)`, `useTranslations(lang)`, `getLocalePath(lang, path)`
- `src/config/seo.sv.ts` — Swedish SEO titles + descriptions for all pages
- Every `.astro` component derives lang via `getLangFromUrl(Astro.url)` — no prop drilling
- `BaseLayout.astro` sets `<html lang={lang}>` dynamically

---

## Content rules

- **Always add strings to `en.ts` first**, then immediately add to `sv.ts` — never leave them out of sync
- `sv.ts` uses `satisfies typeof en` — TypeScript will fail the build if any key is missing
- **Never hardcode user-facing strings in components** — all copy goes through `useTranslations`
- When adding a new page, add SEO metadata to both `src/config/seo.ts` and `src/config/seo.sv.ts`

---

## Page mirroring (hard rule)

**Whenever a section is added to or removed from an English page, apply the same change to its Swedish mirror in `src/pages/sv/`.** Swedish pages are never updated automatically. No section change is complete until both EN and SV files are updated.

When adding a new Swedish page, verify it is structurally equivalent to its English counterpart.

## Layout adaptations for translated content

Translated text (Swedish, and most other languages) can run 20–40% longer than English. This causes layout problems in constrained containers.

### Rules

**Never use fixed heights on text containers.**
Use `min-h-*` if needed, never `h-*` on elements containing translated text.

**Never use `<br>` for layout line breaks.**
Forced breaks look wrong in other languages. Use `max-w-*` on the container to control line length instead.

**Check both locales after every component change.**
After touching any section or component, verify `/` and `/sv/`. Look for:
- Button/badge labels that overflow or wrap unexpectedly
- Nav items that push the header layout
- Hero headlines significantly longer in SV (may need smaller `text-*` or narrower `max-w-*`)
- Card grids where uneven text lengths break visual alignment

**Hyphenation.**
`lang` is set correctly on `<html>` via `BaseLayout.astro`. Reinforce with `hyphens-auto` on long-form text blocks where overflow is likely.

**Never create locale-specific component variants.**
Fix the component to be locale-agnostic (flexible layout, natural wrapping). Only use locale-specific CSS when there is truly no layout-neutral solution.

## Header language switcher

The header includes a language switcher using `flag-icons` (`fi fi-se`, `fi fi-gb`). It reads `alternateHref` passed to `BaseLayout` — always pass this on detail pages where slugs may differ between locales, or to fall back to the collection index.
